🔥 إنشاء مشروع Firebase جديد لحل مشاكل الاتصال

⚠️ المشكلة الحالية:

مشروع tank-tools لا يمكن الوصول إليه بسبب:

  • Firebase Rules غير مطبقة صحيح
  • صلاحيات API Key منتهية
  • مشاكل في إعدادات المشروع

🏗️ الخطوة 1: إنشاء مشروع جديد

1. اذهب إلى Firebase Console

2. اضغط على "Add project" (إضافة مشروع)

3. اكتب اسم المشروع: knpc-tank-tools-2025

4. فعّل Google Analytics (اختياري)

5. اضغط "Create project"

🗃️ الخطوة 2: إعداد Firestore

1. من القائمة الجانبية، اختر "Firestore Database"

2. اضغط "Create database"

3. اختر "Start in production mode"

4. اختر الموقع الأقرب: us-central1

🛡️ الخطوة 3: إعداد Firebase Rules

1. اذهب إلى "Firestore Database" > "Rules"

2. الصق هذا الكود:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if true; } } }

3. اضغط "Publish"

⚙️ الخطوة 4: الحصول على إعدادات المشروع

1. اذهب إلى "Project settings" (إعدادات المشروع)

2. في تبويب "General"، انزل إلى "Your apps"

3. اضغط على أيقونة "" (Web app)

4. اكتب اسم التطبيق: KNPC Tank Tools

5. انسخ الـ config object

🔄 الخطوة 5: تحديث الكود

بعد الحصول على الـ config الجديد، سأحدث الكود في index.html

أرسل لي الـ Firebase config الجديد وسأحدث الموقع فوراً! 🚀

📊 الخطوة 6: إضافة بيانات الخزانات

بعد تحديث الكود، سأضيف جميع بيانات الخزانات تلقائياً إلى المشروع الجديد

🔀 البديل السريع: إصلاح المشروع الحالي

إذا كنت تريد إصلاح المشروع الحالي بدلاً من إنشاء جديد:

  1. اذهب إلى Firebase Rules للمشروع الحالي
  2. تأكد من نشر الـ Rules الجديدة
  3. تحقق من وجود بيانات في Firestore Database
